Public setup for signup, login, and posting

This app now works in two modes:

1. Local mode
   - Runs on your Mac.
   - Saves accounts/posts to data/db.json.

2. Public mode
   - Runs on Railway/Render/VPS.
   - Set HOST=0.0.0.0 so the public web can reach it.
   - Set DB_FILE to a persistent disk path so accounts/posts stay saved.

Railway setup:

1. Upload this folder to GitHub.
2. Create a Railway project from the GitHub repo.
3. Railway will use the included Dockerfile/railway.json.
4. Add a Volume to the service.
5. Mount the volume at:
   /data
6. Add these Variables:

   NODE_ENV=production
   HOST=0.0.0.0
   DB_FILE=/data/vaultmarket-db.json
   OWNER_USERNAME=zaso
   OWNER_EMAIL=your-email@example.com
   OWNER_PASSWORD=make-a-real-password
   SOL_WALLET=AE6vaxpfmPDtJNd1e5oboN5uZFqVYJMuwDyqykrCADvY
   BTC_WALLET=bc1q4h9qnd5slacywkl87umlzxe9zxnpjjjzrjyed2
   LTC_WALLET=LKhmv1GteaCj2eNREN9iMYdZNzbzDo2Gap
   ETH_WALLET=0xb446020017eCb21F3ffE3DED59c770cFA0A1A96F

7. Start command:
   npm start

Render setup:

1. Upload this folder to GitHub.
2. Create a new Render Blueprint from the repo.
3. Render will use render.yaml.
4. Set OWNER_PASSWORD to a real password before launch.

Important:

- Signup, login, and posting will work publicly once the app is deployed with a persistent DB_FILE.
- Change OWNER_PASSWORD before the first public launch.
- If a db file already exists, changing OWNER_PASSWORD will not update the existing admin account. Delete the old DB file or change the password through code/admin flow.
- This is the simplest public setup. Later, move DB_FILE storage to Postgres when the site starts getting real usage.
